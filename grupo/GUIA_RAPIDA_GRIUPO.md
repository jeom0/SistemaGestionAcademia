# 🚀 GUÍA RÁPIDA: CÓMO SUBIR EL PROYECTO (3 PASOS)

Esta es la forma más simple de terminar el proyecto en grupo. Sigan este orden para que no se borren el trabajo entre ustedes.

---

### 🟢 PASO 0: LO QUE YA DEBE ESTAR EN GITHUB
Antes de que nadie suba nada, el repositorio `https://github.com/jeom0/SistemaGestionAcademia.git` debe tener el proyecto Laravel base instalado. Si está vacío, el **Integrante 1** debe subir el proyecto primero.

---

### 1️⃣ PRIMER TURNO: INTEGRANTE 1 (DISEÑO Y BASE)
**¿Qué subes?**: La apariencia de la página (colores, menús) y la Auditoría.
1. Copia tus archivos de la carpeta `01_integrante_1` a tu proyecto.
2. Abre la terminal y escribe:
   ```bash
   git add .
   git commit -m "Diseño y Base"
   git push origin main
   ```

---

### 2️⃣ SEGUNDO TURNO: INTEGRANTE 2 (PERFIL Y PAGOS)
**¿Qué subes?**: Las fotos de perfil, los Dashboards y lo de Comisiones/Descuentos.
1. **IMPORTANTE**: Primero haz un `git pull` para bajar lo que subió el Integrante 1.
2. Copia tus archivos de la carpeta `02_integrante_2` a tu proyecto.
3. Escribe en la terminal:
   ```bash
   git add .
   git commit -m "Perfil y Pagos"
   git push origin main
   ```

---

### 3️⃣ TERCER TURNO: INTEGRANTE 3 (USUARIOS Y PLATA)
**¿Qué subes?**: El registro de usuarios y los ingresos/egresos de dinero.
1. **IMPORTANTE**: Primero haz un `git pull` para bajar lo que subieron los otros dos.
2. Copia tus archivos de la carpeta `03_integrante_3` a tu proyecto.
3. Escribe en la terminal:
   ```bash
   git add .
   git commit -m "Usuarios y Movimientos"
   git push origin main
   ```

---

## 💡 REGLA DE ORO
**Antes de subir tus archivos, siempre haz `git pull`**. Esto evita que borres el trabajo de tus compañeros.
